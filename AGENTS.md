## 项目定位

`cotton-filter` 是 Tauri + React + Python 的本地桌面工具，用于从 Excel 棉花资源表中筛选符合规则的批次。客户仍以 Excel 为主，不要默认改成必须部署数据库或服务端。

## 主要入口

- `backend/server.py`：FastAPI 本地后端入口。
- `cotton_filter_app/rules.py`：SQLite 规则库、默认规则、导入导出和迁移。
- `cotton_filter_app/header.py`：表头识别和列映射。
- `cotton_filter_app/scoring.py`：评分和颜色级占比解析。
- `cotton_filter_app/processor.py`：sheet/workbook 处理、筛选和输出。
- `cotton_filter_app/file_utils.py`：批量文件、输出路径和重名递增。
- `src/App.tsx`：文件筛选主界面。
- `src/RulesView.tsx`：规则维护界面。
- `src/backend.ts`：Tauri 启停 Python sidecar。
- `src/api.ts`：前端调用本地 API。

## 当前处理规则

- 只支持 Tauri GUI，不新增命令行入口。
- 支持 `.xlsx` 和 `.xls`。
- 必需字段是 `基差`、`长度`、`马值`。
- 自动扫描前置行定位真实表头；列名规则只做标准化后的精确匹配。
- 支持两行组合颜色级表头，例如 `颜色级比例(%)` 下的 `白棉2级`、`白棉3级` 会合并成 `颜色级:占比%` 文本。
- 颜色级支持 `95%`、`95％`、`31:84.4` 等写法；单独写 `白棉3级` 或 `31` 时按值别名视为该级别 `100%`。
- 颜色级区间按规则“适用值”的所有别名解析占比。例如适用值为 `白棉3级` 时，`白棉3级:90.7%` 和 `31:90.7%` 都按 `白棉3级` 判断。
- 数值字段支持从 `30.5mm`、`4.1(A)`、`打包-1500` 等文本中提取数字。
- 每行计算 `得分`，再计算 `与基差差距 = 基差 - 得分`。

## 数据规则

- 默认只初始化列名规则和颜色级值别名，不初始化评分区间或过滤区间。
- 值别名只用于 `颜色级`。
- 颜色级值别名区分“匹配值”和“输出值”：如 `31`、`白棉三级`、`三级棉` 都输出为 `白棉3级`。
- 长度、强力、马值、整齐度等数值字段不维护值别名，按数值区间直接判断。
- 颜色级评分区间和过滤区间必须选择颜色级值别名输出值，不能留空或自由填写。
- 非颜色级数值区间不显示、不保存“适用值”，直接按最小值/最大值判断。
- 同一字段下多条过滤区间是“或”；不同字段之间是“且”。
- 关键词过滤是排除规则：字段文本包含关键词时，该行直接过滤出去，不再校验其它区间规则。
- 启动迁移会删除 `批号`、`仓库` 的列名规则和数据规则，删除非颜色级值别名，并清空非颜色级区间规则上的旧适用值。

## 规则维护界面

- 顶部菜单只保留“文件筛选”和“规则维护”。
- 规则维护包含“列名规则”“值别名”“评分与过滤区间”。
- 列名规则页不展示 `批号`、`仓库`；支持新增标准字段，并为该字段写入首个列名别名。
- 值别名页只展示和维护 `颜色级`。
- 评分与过滤区间页不展示 `批号`、`仓库`；动态新增的标准字段也可配置数值区间或关键词过滤。
- 数据规则不暴露“规则名称”；后端只保留兼容字段。
- 支持一键导出/导入全部规则；导入会替换当前规则，并跳过不再维护的字段。

## 文件筛选与输出

- 只通过文件选择入口添加 Excel 文件，不支持拖拽导入。
- 每次“添加 Excel”都对本次所选文件立即筛选；即使重复选择同一文件，也要重新生成结果。
- 前端不缓存、不维护累计文件列表，也不做历史文件去重。
- 不要让生成结果再次被输入扫描处理。
- 输出目录为 `cotton-filter-results`。
- 输出文件名使用原文件名；重名时追加 `_1`、`_2` 等，绝不覆盖已有结果。
- 即使没有命中行，也要写出空结果工作簿，便于确认筛选已执行并触发重名递增。
- 输出工作簿包含 `筛选结果` 和 `识别异常`。
- `筛选结果` 使用原始 Excel 列名和原始单元格数据，并追加 `得分`、`与基差差距`。
- `识别异常` 只展示需要处理的问题：缺少必需字段、疑似标准字段但未配置列名规则、基差无法解析、颜色级值未命中值别名等。

## 本地规则库

- 用户规则保存在本机用户数据目录的 `rules.sqlite3`。
- 可通过 `COTTON_FILTER_RULE_DB` 指定规则库路径。
- 正常自动更新或覆盖安装新版应用不应删除规则库。
